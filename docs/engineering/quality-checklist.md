# Quality Checklist

## Code
- Code must be readable without external explanation
- Each file has a single responsibility
- No duplicated logic
- No dead code

## Architecture
- Structure is clear before implementation
- No unnecessary coupling between modules

## Documentation
- Every project includes a README
- Purpose and structure are explicitly defined

## Git
- Each commit has a clear intention
- Messages follow convention (feat:, docs:, refactor:)
- No vague commits like "update"
