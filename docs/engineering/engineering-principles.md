# Engineering Principles v1

_Author: Angela Jasso_ 
_Date: 2026-03-30_

---

## 1. Design Before Code
No implementation without a defined system structure.

## 2. Single Responsibility
Each module, file, or function has one clear purpose.

## 3. Structure First
System organization takes priority over delivery speed.

## 4. Mandatory Clarity
If it cannot be understood quickly, it is poorly designed.

## 5. No Unnecessary Coupling
Modules must not depend on internal details of others.

## 6. Atomic Commits
Each commit represents a single clear intention.

## 7. No Dead Code
Nothing remains in the system without active purpose.

## 8. Refactoring Is Mandatory
Complexity does not accumulate.

## 9. Maintenance-Oriented Decisions
Every technical decision considers long-term impact.

## 10. Nothing Improvised Reaches Main
The main branch is always stable and clean.

## 11. Readability Over Cleverness
Clear and predictable code is preferred over clever or complex solutions.

## 12. Naming Is Architecture
Names must describe intent, not implementation.

## 13. Controlled Complexity
Complexity must be isolated and never spread across the system.
