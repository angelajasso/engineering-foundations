## Design System Summary

A **Design System** is a collection of reusable visual rules used to maintain consistency across a user interface.

Instead of using arbitrary values for styles, professional interfaces define **design tokens** such as colors, spacing, and typography. These tokens allow styles to be reused and updated globally.

The main foundations of a design system include:

- **Typography scale** to create a clear visual hierarchy.
- **Spacing scale** to maintain consistent layout rhythm.
- **Color tokens** to define semantic roles like primary, background, and text.
- **CSS variables** to centralize and reuse styles across the application.

By combining these elements, developers can build **reusable UI components** such as buttons, cards, and layouts.

Design systems improve:

- consistency
- maintainability
- scalability

Instead of designing individual pages, modern frontend development focuses on **designing reusable systems of rules** that ensure a cohesive interface across an entire product.
