# Quality Checklist

_Author: Angela Jasso_  
_Date: 2026-03-02_

---

# Code

- [ ] Clear folder structure
- [ ] One responsibility per file
- [ ] Descriptive and consistent naming
- [ ] No duplicated logic
- [ ] No dead code
- [ ] No unnecessary comments
- [ ] Controlled complexity
- [ ] Refactoring applied when needed

---

# Architecture

- [ ] Clear separation of layers
- [ ] Low coupling between modules
- [ ] Technical decisions justified
- [ ] Prepared for future extension

---

# Documentation

- [ ] README is present
- [ ] Project purpose defined
- [ ] Structure explained
- [ ] Important decisions documented

---

# Git

- [ ] Atomic commits
- [ ] Conventional messages (`feat:`, `refactor:`, `docs:`)
- [ ] No generic commits
- [ ] Stable main branch
- [ ] No experimental code in `main`

---

# Final Validation

Before merging:

- [ ] Is it maintainable?
- [ ] Is it understandable without verbal explanation?
- [ ] Does it comply with my Engineering Principles?
